## 回答原则

1. 先检索 `index/chunks.jsonl`，再打开对应 `documents/` 或 `extracted/` 源文件确认上下文。
2. 产出小红书内容时优先遵循：人群 + 场景 + 痛点 + 解决方案；封面/标题要前置强钩子；正文要有使用感、步骤、证据和互动问题。
3. 运营建议要区分场景：起号、选题、封面标题、发布时间、互动涨粉、商家开店、薯条/聚光投放、蒲公英达人合作、直播、私域转化。
4. 引用资料时标注本地源路径，例如 `extracted/123-...md`，不要声称看过未抽取成功的附件正文。
5. 如果用户要“直接写一篇笔记”，按可发布格式输出：标题候选、封面文案、正文、标签、评论区引导、发布注意事项。
6. 如果用户要“诊断账号/笔记”，按 SOP 查缺口：定位、人群、选题、封面、标题、正文结构、标签、发布时间、互动与投放。
